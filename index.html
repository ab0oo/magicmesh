Help

A Meshtastic-inspired flood routing + LoRa timing simulator (visual + educational, not a full protocol implementation).

What you’re seeing

  • Nodes are radios. Links show likely neighbors based on range + LOS.
  • Floods are broadcast messages; nodes rebroadcast with duplicate suppression.
  • Pulses visualize on-air transmissions expanding to the node’s radio range.

What’s simulated (Meshtastic-inspired)

  • TTL / max hops: nodes can receive at max hop but won’t relay past TTL.
  • LoRa on-air time: computed from Semtech ToA math using the selected preset + packet size.
  • LBT/CAD-style behavior: nodes defer if the channel is busy and retry with backoff.
  • Node roles: Router (star), Client (circle), Mute (triangle; receives but never retransmits).
  • Curvature LOS: spherical-earth “radio horizon” using node height AGL.
  • Transmit power: each node has a TX power (dBm) that affects its effective range (capped at 40 dBm).
  • Path loss (n): controls how quickly signal falls off with distance; larger n reduces effective range.

What it’s not

  • No real packet formats, encryption, or channel plans.
  • RF propagation is simplified (range + curvature LOS gating; no diffraction/refraction).

Controls

  • Click node: inject a flood.
  • Shift-click: pin/unpin a node.
  • Drag node: reposition.
  • Right-click (or Alt/Ctrl/Cmd+Click): set node height (AGL).
  • Drag from palette (lower-left on canvas): add a node.
